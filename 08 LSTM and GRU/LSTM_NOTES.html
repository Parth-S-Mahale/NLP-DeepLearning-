Understanding LSTM Networks

📄 Source: Olah, C. (2015) — colah.github.io 🏷️ Deep Learning · Sequence Modelling · RNNs
§1 — Recurrent Neural Networks (RNNs)

Standard feedforward neural networks process each input independently, with no memory of what came before. But for sequences — sentences, time series, video frames — context from previous steps is essential. Recurrent Neural Networks (RNNs) solve this by passing a hidden state from one step to the next, giving the network a form of memory.

Thinking of an RNN as a looping box can seem mysterious. But "unrolling" the loop reveals it's simply the same network applied repeatedly, with each copy passing its output to the next. This chain structure makes RNNs the natural architecture for sequential data.

An RNN with its loop "unrolled" — the same module A is applied at each timestep, passing hidden state hₜ to the next step
Key intuition: Think of the hidden state hₜ as a "working memory" note that gets updated at every step and passed forward. Each module reads both the current input xₜ and the previous memory hₜ₋₁.

RNNs have achieved impressive results in speech recognition, language modelling, machine translation, and image captioning. But vanilla RNNs suffer from a critical limitation with longer sequences.

§2 — The Problem of Long-Term Dependencies

Sometimes nearby context is enough. Predicting the last word in "the clouds are in the ___" only needs the last few words. But consider: "I grew up in France… I speak fluent ___." Here the relevant context (France) is many steps back.

Vanishing gradient: relevant context from far back (highlighted in red) cannot reliably reach the output in a standard RNN
The vanishing gradient problem: Backpropagation through time multiplies gradients repeatedly. If the values are slightly less than 1, repeated multiplication drives the gradient exponentially toward zero. The network simply cannot learn connections across long gaps.

In theory, RNNs can handle long dependencies — you can hand-pick parameters to do so. In practice, it's been shown (Hochreiter 1991, Bengio et al. 1994) that standard RNNs reliably fail to learn them. LSTMs were designed specifically to solve this.

§3 — LSTM Networks: Introduction

Long Short-Term Memory networks (LSTMs), introduced by Hochreiter & Schmidhuber (1997), are a special kind of RNN built to avoid the vanishing gradient problem. Remembering information across long sequences is essentially their default behaviour.

Like all RNNs, LSTMs have a chain of repeating modules. But where a standard RNN module contains a single layer (typically tanh), an LSTM module contains four interacting neural network layers.

Standard RNN module (left) vs. LSTM module (right). The LSTM adds a dedicated "cell state" highway at the top and replaces the single layer with four coordinated layers.
Notation guide: In LSTM diagrams — yellow/green boxes are learned neural network layers; pink circles are pointwise operations (add or multiply); merging lines = concatenation; forking lines = copy.
§4 — Core Idea: The Cell State

The magic of the LSTM is its cell state (Cₜ) — the horizontal line running across the top of the module. Think of it as a conveyor belt: information flows straight along it with minimal interference, making it very easy for gradients to propagate unchanged across hundreds of timesteps.

The cell state as a conveyor belt — information flows unimpeded, with gates selectively removing (×) or adding (+) information

The LSTM regulates information flow on the cell state using gates. Each gate is a sigmoid layer (σ) that outputs values between 0 and 1, multiplied pointwise with the data. A value of 0 means "block everything"; a value of 1 means "pass everything through."

A gate is a sigmoid layer followed by pointwise multiplication — the sigmoid "decides" how much of the signal to let through

An LSTM has three gates: the Forget gate decides what to erase, the Input gate decides what to write, and the Output gate decides what to read.

§5 — Step-by-Step LSTM Walkthrough

Let's walk through the four operations inside an LSTM cell using a language model example (predicting the next word) to build intuition for each gate's role.

1

Forget Gate — What to throw away

The first step decides what information to discard from the cell state. A sigmoid layer called the "forget gate" looks at the previous hidden state hₜ₋₁ and the current input xₜ, and outputs a number between 0 and 1 for every value in the cell state. 1 = keep completely, 0 = forget completely.

Step 1 — Forget gate: highlighted in red. The σ layer produces fₜ ∈ [0,1] for each cell value
fₜ = σ(Wf · [hₜ₋₁, xₜ] + bf) // σ squashes to (0,1); fₜ close to 0 forgets, close to 1 keeps
2

Input Gate — What new information to store

This step has two parts: (a) a sigmoid layer (the "input gate") decides which values to update; (b) a tanh layer creates a vector of candidate values C̃ₜ that could be added. These are combined in the next step.

Step 2 — Input gate (σ) + Candidate values (tanh): highlighted in green. Their product is added to the cell state.
iₜ = σ(Wᵢ · [hₜ₋₁, xₜ] + bᵢ) // input gate: which values to update C̃ₜ = tanh(W_C · [hₜ₋₁, xₜ] + b_C) // candidate values (range: -1 to 1)
3

Cell State Update — Combining forget and input

Now we perform the actual cell state update. We multiply the old cell state by fₜ (forgetting what we decided to forget), then add iₜ × C̃ₜ (scaling the new candidates by how much we decided to update each position).

Step 3 — Cell state update: old state scaled by forget gate, then new candidates scaled by input gate are added
Cₜ = fₜ * Cₜ₋₁ + iₜ * C̃ₜ // fₜ*Cₜ₋₁ = selectively forget; iₜ*C̃ₜ = selectively write new info
4

Output Gate — What to read out as hidden state

The final step decides what to output. A sigmoid layer (output gate) decides which parts of the cell state to expose. The cell state is then pushed through tanh (to squash values to [-1,1]) and multiplied by the output gate — giving us the new hidden state hₜ.

Step 4 — Output gate: highlighted in purple. The cell state passes through tanh and is filtered by the output gate σ to produce hₜ
oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o) // output gate hₜ = oₜ * tanh(Cₜ) // new hidden state (what we expose)
Why this works: The cell state Cₜ can carry relevant information over many steps with minimal modification (only the small × and + operations on it). Gradients flow cleanly backward through the + operation — there's always a direct gradient path, no repeated matrix multiplication shrinking it to zero.
§6 — Variants on LSTM

In practice, almost every paper that uses LSTMs uses a slightly modified version. The differences are often minor, but three variants are worth knowing.

VariantWhat changesWhy it matters
Peephole Connections
Gers & Schmidhuber, 2000		 Gate layers can "peek" at the cell state Cₜ₋₁ directly (in addition to hₜ₋₁ and xₜ) Useful when precise timing matters (e.g. identifying exact intervals in a sequence)
Coupled Forget/Input Instead of separate forget and input decisions, they are tied: we only input when we forget, and vice versa Simpler model; prevents wasting capacity storing redundant info
GRU
Cho et al., 2014		 Merges forget+input into one "update gate"; merges cell state and hidden state; removes separate output gate Simpler (fewer parameters), often same performance as LSTM, faster to train. Increasingly popular.

Greff et al. (2015) compared eight major LSTM variants systematically and found they all perform approximately the same on most tasks. The vanilla LSTM remains an excellent default choice.

GRU collapses the LSTM's 3 gates into 2 (update + reset) and removes the separate cell state. Simpler but often equally powerful.
zₜ = σ(W_z · [hₜ₋₁, xₜ]) // update gate (combined forget+input) rₜ = σ(W_r · [hₜ₋₁, xₜ]) // reset gate h̃ₜ = tanh(W · [rₜ * hₜ₋₁, xₜ]) // candidate hidden state hₜ = (1 − zₜ) * hₜ₋₁ + zₜ * h̃ₜ // interpolate old + new

Gate / StepEquationRole
Forget fₜ σ(Wf·[hₜ₋₁, xₜ] + bf) What % of old cell state to keep
Input iₜ σ(Wᵢ·[hₜ₋₁, xₜ] + bᵢ) Which positions to update

The big picture in one sentence: An LSTM cell maintains two memory streams — a cell state Cₜ (long-term, lightly modified each step) and a hidden state hₜ (short-term, fully recomputed each step) — connected by three learned gates that decide what to remember, update, and output at each timestep.
